# PROJECT

## 목적
이 프로젝트는 Raspberry Pi 환경에서 장기간 안정적으로 동작하도록 설계된 개인용 시장 데이터 수집기입니다. 핵심은 한 번에 많이 받는 것보다, 매일 보수적으로 수집하고 중단되어도 다시 이어서 실행할 수 있게 만드는 것입니다.

## 핵심 설계 원칙
- **하루에 다 못 끝내도 정상**: 한 번의 실행이 모든 심볼을 처리하지 못해도 정상으로 간주합니다.
- **API 제한 최우선**: provider별 일일 한도와 reserve를 남기는 보수적 예산 사용을 기본으로 합니다.
- **상태 파일 기반 재개**: `data/state/state.json`을 기준으로 provider 사용량과 심볼 처리 상태를 기록/재개합니다.

## 절대 하지 말아야 할 것
- **API 한도 초과 유도 금지**: 일일 예산/리저브 원칙을 깨는 설정 또는 코드 변경 금지.
- **복잡한 파티셔닝 도입 금지**: Stage 1 범위를 넘어서는 과도한 데이터 분할 구조 도입 금지.
- **외부 의존성 무분별 추가 금지**: 표준 라이브러리 중심 MVP 원칙을 깨는 불필요 의존성 추가 금지.
